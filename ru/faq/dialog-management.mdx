## УПРАВЛЕНИЕ ДИАЛОГАМИ И ИСТОРИЯ

<AccordionGroup>
  <Accordion title="Как очистить историю/память бота?">
    Отправьте в чат комбинацию: 😎😇😭😚😍. Либо откройте ЛК → раздел «Чаты» → выберите диалог → нажмите «Очистить историю». После очистки ИИ начинает диалог с нуля, без учёта предыдущего контекста.
  </Accordion>
  <Accordion title="После очистки истории ИИ всё равно «помнит» старые данные.">
    Обратите внимание на особенность поведения: очистка истории диалога и очистка профиля гостя — это разные операции. Данные профиля (имя, предпочтения) сохраняются отдельно. Функция полной очистки профиля гостя находится в разработке. Если нужно сбросить профиль — напишите в поддержку.
  </Accordion>
  <Accordion title="Можно ли настроить автоматическую очистку через X часов?">
    Да. Откройте ЛК → настройки канала → найдите пункт «Автоочистка чата» → задайте нужный интервал в часах. Изменения применяются сразу.
  </Accordion>
  <Accordion title="ИИ начинает диалог заново с приветствия посреди разговора.">
    Проверьте настройку автоочистки в ЛК → настройки канала. Если интервал слишком короткий, ИИ сбрасывает контекст раньше, чем завершается диалог. Увеличьте интервал до комфортного значения.
  </Accordion>
  <Accordion title="Где смотреть историю переписки ИИ с клиентами?">
    Откройте ЛК → раздел «Чаты». Здесь отображаются все диалоги по всем сотрудникам и каналам. Используйте фильтры по сотруднику и по каналу для быстрого поиска нужного диалога.
  </Accordion>
  <Accordion title="Как поделиться ссылкой на конкретный диалог?">
    Откройте ЛК → раздел «Чаты» → выберите нужный диалог → скопируйте URL из адресной строки браузера. Ссылка имеет формат: https://app.aitextura.com/chat/[ID]. Это удобнее скриншотов: получатель сразу видит полный контекст.
  </Accordion>
  <Accordion title="Как переобучить ИИ после изменений в Google Docs?">
    После сохранения изменений в Google Docs выполните два шага:
    1. Отправьте в любой чат команду переобучения: 📚📕📙📗📓📖. Либо нажмите «Сохранить / Переобучить» в ЛК.
    2. Сразу после переобучения очистите память сотрудника (😎😇😭😚😍), чтобы ИИ начал использовать новые данные с первого сообщения.
  </Accordion>
</AccordionGroup>

## ВМЕШАТЕЛЬСТВО ОПЕРАТОРА / ПАУЗА ИИ

<AccordionGroup>
  <Accordion title="Что происходит, когда менеджер вручную отвечает в чате?">
    ИИ автоматически переходит на паузу — по умолчанию на 15–20 минут. По истечении паузы ИИ возобновляет работу в этом чате самостоятельно. Длительность паузы настраивается.
  </Accordion>
  <Accordion title="Можно ли сделать, чтобы ИИ не возвращался после ответа менеджера?">
    Да. Откройте ЛК → настройки сотрудника → профессиональные настройки → отключите опцию «Отвечать на сообщения, полученные во время паузы». В этом режиме после ответа менеджера ИИ полностью отключается для данного чата и не возобновляет работу автоматически.
  </Accordion>
  <Accordion title="Как отключить ИИ в конкретном чате?">
    Откройте ЛК → раздел «Чаты» → выберите нужный диалог → найдите переключатель ИИ → переведите в режим паузы. ИИ перестанет отвечать только в этом чате; остальные диалоги не затрагиваются.
  </Accordion>
  <Accordion title="Как полностью отключить ИИ-сотрудника?">
    Откройте ЛК → список сотрудников → найдите нужного сотрудника → нажмите на зелёный переключатель рядом с его именем. Переключатель станет серым — сотрудник деактивирован во всех каналах.
  </Accordion>
  <Accordion title="Как настроить стоп-слово?">
    Откройте настройки нужного сотрудника → найдите поле «Стоп-слово» → введите нужную фразу. Примеры: «Алина, остановись» или три запятые (,,,). Важный нюанс: не используйте в качестве стоп-слова часто встречающиеся слова вроде «Здравствуйте» — ИИ будет останавливаться при каждом приветствии гостя.
  </Accordion>
  <Accordion title="Можно ли регулировать время паузы?">
    Да. Время паузы настраивается в настройках сотрудника (если опция доступна в вашем тарифе) или через поддержку. По умолчанию установлено 15–20 минут.
  </Accordion>
  <Accordion title="Как включить ИИ обратно после паузы?">
    Два способа:
    1. Отправьте в нужный чат символ 😇 — ИИ возобновит работу немедленно.
    2. Откройте ЛК → раздел «Чаты» → выберите диалог → включите переключатель ИИ вручную.
  </Accordion>
</AccordionGroup>

## УВЕДОМЛЕНИЯ И ТРИГГЕРЫ

<AccordionGroup>
  <Accordion title="Как настроить уведомления менеджерам?">
    Настройте уведомления самостоятельно в несколько шагов:
    1. Войдите в Личный кабинет → раздел «Уведомления».
    2. Нажмите «Получать уведомления» — система сформирует ссылку на активацию бота @aitexturanotificationbot в Telegram.
    3. Каждый менеджер переходит по ссылке и самостоятельно активирует бот у себя в Telegram.
    Уведомления начнут поступать сразу после активации.
  </Accordion>
  <Accordion title="Какие триггеры доступны?">
    Доступны 4 базовых триггера: «Нужен человек», «Подтверждение бронирования», «Жалоба», «Возврат средств». Чтобы добавить собственный триггер: перейдите в раздел «Триггеры» → нажмите «Добавить» → опишите условие на естественном языке. ИИ будет распознавать его автоматически.
  </Accordion>
  <Accordion title="Уведомления не приходят.">
    Выполните проверку по шагам:
    1. Убедитесь, что бот @aitexturanotificationbot активирован в Telegram именно на вашем аккаунте — пройдите по ссылке из ЛК повторно.
    2. Откройте настройки триггера и проверьте, что флаг «Стоп» не активирован — при включённом «Стопе» уведомления намеренно блокируются.
    3. Проверьте статус платформы на status.aitextura.com: при нестандартном поведении канала уведомлений мы анализируем цепочку запросов и восстанавливаем доставку.
    Если все три пункта в порядке, а уведомления по-прежнему не поступают — обратитесь в поддержку с указанием ID триггера и временного диапазона.
  </Accordion>
  <Accordion title="Можно ли отправлять уведомления на почту / вебхук?">
    Да. Настройте интеграцию через вебхук (HTTP POST):
    1. В разделе «Уведомления» выберите тип доставки «Вебхук».
    2. Укажите URL вашего эндпоинта.
    3. При необходимости добавьте заголовки авторизации.
    Вебхук совместим со Slack, Notion, Make, Zapier и любым сервисом, принимающим HTTP POST.
  </Accordion>
  <Accordion title="В уведомлении будет ссылка на переписку?">
    Да. Каждое уведомление содержит прямую ссылку на соответствующий диалог в ЛК — вы сразу попадаете в нужный чат без поиска.
  </Accordion>
  <Accordion title="Уведомления приходят на каждое сообщение — как настроить только важные?">
    Скорректируйте формулировки триггеров:
    1. Откройте раздел «Триггеры» и просмотрите каждый активный триггер.
    2. Уберите широкие или расплывчатые формулировки, которые срабатывают на большинство сообщений.
    3. Сформулируйте условие максимально конкретно: укажите ключевые слова, тему или намерение гостя.
    4. Сохраните изменения и протестируйте на 10–15 реальных диалогах.
    После корректировки уведомления будут поступать только по целевым ситуациям.
  </Accordion>
</AccordionGroup>

## БУФЕР СООБЩЕНИЙ И СКОРОСТЬ ОТВЕТА

<AccordionGroup>
  <Accordion title="ИИ отвечает на каждое сообщение отдельно, если гость пишет несколько подряд.">
    Это стандартное поведение без буфера. Чтобы ИИ дожидался всех сообщений перед ответом:
    1. Откройте настройки нужного канала.
    2. Найдите параметр «Буфер сообщений».
    3. Установите задержку 5–10 секунд.
    4. Сохраните. Теперь ИИ будет накапливать входящие сообщения в течение заданного времени и отвечать единым развёрнутым ответом.
  </Accordion>
  <Accordion title="Гость не успевает дописать — ИИ уже отвечает.">
    Увеличьте значение буфера в настройках канала:
    1. Настройки → нужный канал → «Буфер сообщений».
    2. Увеличьте задержку — например, с 5 до 10–15 секунд.
    3. Сохраните изменения.
    Подберите значение под характерный ритм общения ваших гостей.
  </Accordion>
  <Accordion title="ИИ отвечает слишком долго (полминуты и более).">
    Увеличенное время отклика может быть связано с несколькими техническими особенностями:
    1. ИИ одновременно обращается к нескольким Талантам — проверьте, не перегружена ли цепочка инструкций, и при необходимости упростите её.
    2. Проверьте текущее состояние платформы на status.aitextura.com — мы анализируем нагрузку на серверы в режиме реального времени.
    3. Оцените нюансы интеграции с конкретным каналом — некоторые каналы имеют собственные ограничения скорости доставки.
    Если замедление носит систематический характер, передайте в поддержку ID канала и временные метки диалогов.
  </Accordion>
  <Accordion title="Можно ли для виджета на сайте отключить буфер?">
    Да. Для веб-виджета скорость реакции критична, поэтому буфер рекомендуется уменьшить или полностью отключить:
    1. Настройки → канал «Виджет на сайте» → «Буфер сообщений».
    2. Установите значение 0 или минимально допустимое.
    3. Сохраните.
  </Accordion>
</AccordionGroup>
