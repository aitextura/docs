## ПРОФИЛИ КЛИЕНТОВ И АНАЛИТИКА

<AccordionGroup>
  <Accordion title="Как работают профили клиентов?">
    ИИ автоматически формирует профиль гостя в процессе диалога. По умолчанию фиксируются имя и язык общения. Чтобы собирать дополнительные данные:
    1. Перейдите в раздел «Профили клиентов» → «Настройки полей».
    2. Добавьте нужные поля (например: тип гостя, предпочтения, дата визита).
    3. В инструкции Таланта укажите, при каких условиях заполнять каждое поле.
  </Accordion>
  <Accordion title="Какие режимы обновления профилей?">
    Доступны два режима — выберите подходящий:
    - **Realtime** — профиль обновляется после каждого сообщения. Подходит для сценариев, где актуальность данных критична. Расходует больше токенов.
    - **Плановая** — обновление по расписанию. Оптимально для аналитических задач, существенно экономит токены.
    Режим настраивается в разделе «Профили клиентов» → «Режим обновления».
  </Accordion>
  <Accordion title="Где смотреть аналитику?">
    Аналитика доступна в нескольких разделах ЛК:
    1. **Дашборд (рабочий стол)** → общие метрики за 24 часа, 7 дней, 30 дней.
    2. **Профили** → статистика заполненности полей по базе гостей.
    3. **Чаты** → откройте любой диалог → напротив сообщения ИИ нажмите «Анализ ответа» для детального разбора конкретного взаимодействия.
  </Accordion>
  <Accordion title="Как посмотреть, какие данные ИИ использовал для ответа?">
    Перейдите в раздел «Чаты» → откройте нужный диалог → найдите сообщение ИИ → нажмите кнопку «Анализ ответа» напротив него. Откроется детальная разбивка: какие источники данных, Таланты и фрагменты инструкций были задействованы при формировании ответа.
  </Accordion>
  <Accordion title="Есть ли бот-суммаризатор (ежедневная сводка)?">
    Да. Для подключения ежедневной сводки:
    1. Добавьте бота @TexturaAi_summary_bot в нужную Telegram-группу с правами администратора.
    2. Бот автоматически формирует и отправляет сводку за предыдущий день каждое утро.
    Дополнительная настройка не требуется.
  </Accordion>
  <Accordion title="Есть ли анализ отзывов с Booking, Google, TripAdvisor?">
    Да. Для запуска анализа отзывов:
    1. Напишите боту @aitextura_reviews_bot и укажите ссылки на ваши страницы на нужных платформах.
    2. Бот выполнит парсинг отзывов и передаст результаты в Google Таблицу.
    3. Ссылку на таблицу вы получите от бота после завершения обработки.
  </Accordion>
</AccordionGroup>

## АВТОДОЖИМ (FOLLOW-UP)

<AccordionGroup>
  <Accordion title="Что такое Автодожим?">
    Автодожим — это инструмент автоматических напоминаний для клиентов, которые перестали отвечать. Вы настраиваете цепочку шагов с нужными задержками и инструкциями для ИИ — и система сама ведёт диалог до получения ответа.
  </Accordion>
  <Accordion title="Как настроить?">
    Вот как это делается:
    1. Откройте настройки нужного сотрудника.
    2. Перейдите в раздел **Автодожим**.
    3. Нажмите «Добавить шаг».
    4. Для каждого шага укажите задержку (в часах или минутах) и инструкцию для ИИ — что именно написать клиенту.
    5. Повторите для каждого следующего шага.
    Как только клиент отвечает — цепочка сбрасывается автоматически.
  </Accordion>
  <Accordion title="Работает ли автодожим круглосуточно?">
    Автодожим работает только в рамках рабочих часов, заданных в расписании сотрудника — это важный нюанс при планировании цепочки. Обратите внимание: отключить автодожим для отдельного чата не получится — настройка применяется ко всем диалогам сотрудника.
  </Accordion>
  <Accordion title="Рекомендации по настройке?">
    Для большинства сценариев оптимальная схема — 2–3 шага с интервалом не менее 2–4 часов между ними. Учитывайте, что каждое отправленное напоминание потребляет токены — это стоит принимать во внимание при планировании длинных цепочек.
  </Accordion>
</AccordionGroup>

## РАБОЧЕЕ ВРЕМЯ

<AccordionGroup>
  <Accordion title="Как настроить рабочее время ИИ?">
    Вот как это делается:
    1. Откройте настройки нужного сотрудника.
    2. Перейдите в раздел **Рабочее время**.
    3. Задайте расписание по дням недели, укажите часовой пояс, при необходимости добавьте перерывы и праздничные дни.
    4. Сохраните изменения.
    По умолчанию сотрудник работает в режиме 24/7.
  </Accordion>
  <Accordion title="Можно ли для разных каналов разное время?">
    Да. Один сотрудник поддерживает одно расписание, поэтому для разных каналов создайте отдельных сотрудников и задайте каждому своё рабочее время. Это стандартная практика при работе с несколькими площадками.
  </Accordion>
  <Accordion title="Можно ли включить ИИ только с 20:00 до 9:00?">
    Да, это реализуется через расписание:
    1. Откройте настройки сотрудника → **Рабочее время**.
    2. Задайте интервал с 20:00 до 9:00 для нужных дней.
    3. Укажите корректный часовой пояс — это критично для точного срабатывания.
    4. Сохраните.
  </Accordion>
</AccordionGroup>
