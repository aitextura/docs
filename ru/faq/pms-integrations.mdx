## ИНТЕГРАЦИЯ С PMS: TRAVELLINE

<AccordionGroup>
  <Accordion title="Как подключить Travelline?">
    Выполните следующие шаги на платформе V2:
    1. Перейдите в раздел «Таланты» → «Доступные навыки».
    2. Найдите Travelline в конце списка и нажмите «Подключить».
    3. Введите ID отеля из вашего личного кабинета Travelline.
    4. Сохраните настройки и проверьте работу на тестовом запросе.
  </Accordion>
  <Accordion title="Где найти ID отеля в Travelline?">
    Войдите в личный кабинет Travelline. В правом верхнем углу в разделе настроек указан «ID отеля» — это несколько цифр, например 243332. Скопируйте это значение для подключения.
  </Accordion>
  <Accordion title="Что ИИ берёт из Travelline?">
    Данные о ценах и доступности номеров. Данные обновляются динамически. ИИ может озвучить цену, проверить наличие и направить ссылку на бронирование.
  </Accordion>
  <Accordion title="Учитываются ли промокоды при запросе цен?">
    Автоматическое применение промокодов при запросе цен не реализовано. Для работы со статическими промокодами добавьте их описание и условия применения в инструкцию ИИ — сотрудник будет сообщать о них гостю. Автоматическое применение промокода непосредственно при бронировании на данный момент недоступно.
  </Accordion>
  <Accordion title="Можно ли настроить автоматическое бронирование (без участия менеджера)?">
    Да, автобронирование через Travelline доступно. ИИ самостоятельно создаёт бронь. Убедитесь, что соответствующий функционал активирован в настройках ИИ-сотрудника.
  </Accordion>
  <Accordion title="ИИ запрашивает гражданство при бронировании — зачем?">
    Это обязательное поле в форме Travelline — техническая особенность интеграции с PMS. ИИ запрашивает его, чтобы корректно передать данные в систему бронирования. Если необходимо скорректировать формулировку запроса гостю — добавьте соответствующую инструкцию в настройки ИИ-сотрудника.
  </Accordion>
  <Accordion title="Можно ли сделать источником брони «ИИ-сотрудник» вместо «сайт»?">
    Да, это реализуется на стороне Travelline. Выполните следующие шаги:
    1. Обратитесь в службу поддержки Travelline.
    2. Запросите добавление AI Textura как отдельного источника бронирований.
    3. После подтверждения со стороны Travelline источник будет отображаться корректно.
  </Accordion>
  <Accordion title="ИИ в выходные говорит «занято» вместо «минимум 2 ночи».">
    Это нюанс инструкции — ИИ воспроизводит формулировку, которой его не обучили. Исправьте поведение следующим образом:
    1. Перейдите в настройки ИИ-сотрудника → раздел «Инструкция».
    2. Добавьте явное правило, например: «В пятницу и субботу бронирование возможно только от 2 ночей. Сообщи гостю об этом правиле, не используй фразы типа "нет мест" или "занято"».
    3. Сохраните инструкцию и проверьте поведение на тестовом запросе.
  </Accordion>
</AccordionGroup>

## ИНТЕГРАЦИЯ С PMS: BNOVO

<AccordionGroup>
  <Accordion title="Как подключить Bnovo?">
    Выполните следующие шаги:
    1. Перейдите в ЛК AI Textura → «Данные» → «Добавить данные».
    2. Выберите Bnovo из списка интеграций.
    3. Введите логин и пароль от вашего аккаунта Bnovo.
    4. Сохраните настройки и проверьте корректность подключения.
  </Accordion>
  <Accordion title="Где найти ID отеля в Bnovo?">
    ID отеля содержится в адресной строке браузера, когда вы находитесь в личном кабинете Bnovo. Пример URL: `https://bnovo.ru/hotel/12345` — ID это `12345`. Скопируйте это значение для использования в интеграции.
  </Accordion>
  <Accordion title="Какой логин/пароль вводить — владельца или сотрудника?">
    Достаточно данных администратора.
  </Accordion>
  <Accordion title="Ошибка «не найдены доступные отели» при подключении.">
    Это типовое расхождение в данных при вводе учётных данных. Выполните следующие шаги:
    1. Скопируйте логин из вашего аккаунта Bnovo — не вводите вручную, чтобы исключить лишние пробелы.
    2. Сделайте то же самое с паролем.
    3. Вставьте значения в поля подключения и повторите попытку.
    Если расхождение сохраняется — обратитесь в поддержку: мы проверим параметры запроса авторизации.
  </Accordion>
  <Accordion title="ИИ показывает тарифы не по названиям, а по ID.">
    Это нестандартное поведение интеграции с Bnovo, которое мы диагностируем на уровне цепочки запросов. Обратитесь в поддержку — мы проанализируем логи и устраним расхождение в отображении данных.
  </Accordion>
  <Accordion title="ИИ берёт цены из недействующих тарифов.">
    Это нюанс настройки видимости тарифов в Bnovo. Выберите один из следующих вариантов:
    1. В личном кабинете Bnovo настройте, какие тарифы отображаются «от стойки» — ИИ будет работать только с ними.
    2. Либо создайте отдельный тариф, например «Нейротариф», специально для ИИ-сотрудника, и укажите его в инструкции.
  </Accordion>
  <Accordion title="ИИ создаёт бронирование с суммой 0 рублей.">
    Это нюанс интеграции на стороне Bnovo, связанный с передачей данных о стоимости. Выполните следующие шаги:
    1. После создания бронирования ИИ вы найдёте корректную сумму в уведомлении от Bnovo — используйте её для проставления вручную.
    2. Обратитесь в поддержку AI Textura — мы проанализируем цепочку запросов и диагностируем поведение передачи стоимости в вашей конфигурации.
  </Accordion>
  <Accordion title="Bnovo API открыло доступ — как использовать?">
    API Bnovo позволяет получать данные — цены и доступность номеров. Создание бронирований через API Bnovo на данный момент недоступно. Если вам нужно автобронирование — рассмотрите альтернативные PMS-интеграции или уточните у поддержки актуальный статус этой функции.
  </Accordion>
</AccordionGroup>

## ИНТЕГРАЦИЯ С PMS: MEWS

<AccordionGroup>
  <Accordion title="Как получить доступ к MEWS API?">
    Доступны два варианта:
    1. Подайте заявку в MEWS на получение production client name.
    2. Используйте существующий client name, если он уже есть.
    В обоих случаях потребуется пройти сертификацию MEWS (Connector API certification). Уточните текущие требования к сертификации на сайте MEWS или у вашего менеджера MEWS.
  </Accordion>
  <Accordion title="MEWS неправильно считает часовой пояс.">
    Это техническая особенность MEWS API: система ожидает время в часовом поясе отеля, а не в UTC. Для корректной работы убедитесь, что в запросах к MEWS явно передаётся время заезда (15:00) и выезда (11:00) в локальном часовом поясе объекта. Если настройка выполнена самостоятельно и расхождение сохраняется — обратитесь в поддержку: мы проверим параметры запросов.
  </Accordion>
  <Accordion title="Как часто обновляются данные из MEWS?">
    Раз в 2 минуты.
  </Accordion>
</AccordionGroup>

## ДРУГИЕ ИНТЕГРАЦИИ И ТАЛАНТЫ

<AccordionGroup>
  <Accordion title="Может ли ИИ читать данные из Google Таблиц?">
    Да, через Талант Google Sheets. Подготовьте таблицу по следующим требованиям: структура должна быть плоской (без вложенных уровней), заголовки столбцов — понятными и однозначными, объединённые ячейки использовать нельзя. После этого подключите Талант в настройках ИИ-сотрудника.
  </Accordion>
  <Accordion title="Может ли ИИ искать информацию в интернете?">
    Да, через Талант WebSearch. Подключите его в настройках ИИ-сотрудника и добавьте в инструкцию явное указание, в каких случаях использовать поиск — это позволит точно управлять поведением сотрудника.
  </Accordion>
  <Accordion title="Есть ли интеграция с Google Calendar?">
    Да, через Таланты. Подключите соответствующий Талант в настройках ИИ-сотрудника.
  </Accordion>
  <Accordion title="Можно ли подключить свой API?">
    Да. Доступны два варианта:
    1. Создайте собственный Талант (Custom) для отправки HTTP-запросов к любому внешнему API.
    2. Используйте SendAPI для настройки кастомных вебхуков.
    Подключение выполняется в разделе «Таланты» в настройках ИИ-сотрудника.
  </Accordion>
  <Accordion title="Может ли ИИ подключаться к погоде?">
    Да. Для корректной работы укажите точный адрес или местоположение объекта в настройках соответствующего Таланта — это обеспечит точность данных о погоде.
  </Accordion>
  <Accordion title="Есть ли интеграция с Ecvi / RealtyCalendar / MeHotel?">
    Интеграции с этими системами прорабатываются индивидуально. Для реализации необходим доступ к API. Если API вашей PMS закрытый — доступен альтернативный вариант: «ИИ-оператор», при котором ИИ взаимодействует с системой через браузер, как обычный пользователь. Обратитесь в поддержку для обсуждения вашего конкретного случая.
  </Accordion>
  <Accordion title="Есть ли интеграция с WeChat?">
    На данный момент интеграция с WeChat недоступна. Следите за обновлениями в разделе новостей AI Textura.
  </Accordion>
  <Accordion title="Нужно ли добавлять IP-адреса в whitelist для API?">
    Да, при работе с внешними API выполните следующие шаги:
    1. Добавьте в whitelist IP-адреса: `136.144.29.235` и `136.144.28.219`.
    2. Также добавьте домен `aitextura.com`.
    3. Убедитесь, что изменения применены на стороне вашей системы.
  </Accordion>
  <Accordion title="Сколько Талантов доступно?">
    Более 3000 через библиотеку External Talents: OpenAI, Anthropic, Shopify, MySQL, PostgreSQL, AWS, Twilio, amoCRM, Bitrix24 и другие. Просмотреть полный список можно в разделе «Таланты» → «Доступные навыки» в вашем личном кабинете.
  </Accordion>
</AccordionGroup>
