</Info>

## Два места работы с знаниями

В AI Textura есть два связанных, но разных раздела для работы со знаниями:

| Раздел | Где находится | Назначение |
|---|---|---|
| **Знания** | Левое меню платформы | Центральное хранилище всех источников по всем сотрудникам |
| **База знаний** | Вкладка в настройках конкретного сотрудника | Выбор, какие источники из хранилища подключить к сотруднику |

Загружать файлы можно из обоих мест, но есть важное различие:

- **Загрузка через вкладку «База знаний» сотрудника** -- файл автоматически попадает в подпапку этого сотрудника внутри **Agents** и сразу подключается к сотруднику.
- **Загрузка через раздел «Знания» в левом меню** -- файл попадает в выбранную вами папку, но **не подключается** автоматически ни к одному сотруднику. Нужно вручную поставить чекбокс во вкладке **База знаний** нужного сотрудника.

## Раздел «Знания» -- центральное хранилище

Раздел **Знания** в левом меню платформы -- это общее хранилище файлов и источников информации по всем сотрудникам. Здесь вы:

- видите полную картину: какие источники загружены и к каким сотрудникам подключены
- организуете информацию по папкам
- добавляете новые источники
- удаляете устаревшие данные

### Интерфейс

Экран разделён на две части:

**Левая панель -- дерево папок.** Иерархическая структура всех знаний. На верхнем уровне -- корневая папка **All knowledge** («Все знания»). Нажмите на любую папку -- в правой части отобразится её содержимое.

**Правая панель -- таблица файлов.** При выборе папки появляется таблица **Knowledge files**:

| Столбец | Что показывает |
|---|---|
| **Название** | Имя файла, URL или текстового источника |
| **Type** | Тип элемента: Folder, Text, File, Web и др. |
| **Updated** | Дата последнего обновления |

Над таблицей расположены:
- **Хлебные крошки** (breadcrumbs) -- навигация по уровням вложенности
- **Фильтр по сотруднику** (All agents) -- показать источники, подключённые к конкретному сотруднику
- **Кнопка добавления** -- создание папок и загрузка источников

У каждого элемента в таблице есть кнопка действий (три точки): переименовать, переместить или удалить.

### Структура папок

По умолчанию система организует знания следующим образом:

```
All knowledge
├── Agents
│   ├── [Имя сотрудника 1]
│   │   ├── файл.pdf
│   │   ├── текст
│   │   └── ...
│   ├── [Имя сотрудника 2]
│   │   └── ...
│   └── ...
├── FAQ
│   └── ...
└── [Ваши папки]
    └── ...
```

- **Agents** -- системная папка. Внутри автоматически создаются подпапки для каждого сотрудника при добавлении источников через вкладку **База знаний** в настройках сотрудника.
- **FAQ** -- папка для записей из раздела [самообучения](/ru/guides/faq), отправленных в базу знаний.
- **Пользовательские папки** -- создавайте на любом уровне для организации по темам, отделам или проектам.

<Tip>
  Структура папок не влияет на работу сотрудника -- он ищет по всем подключённым источникам, независимо от расположения. Папки нужны исключительно для вашего удобства.
</Tip>

### Создание папки

<Steps>
  <Step title="Перейдите в раздел «Знания»">
    Откройте раздел **Знания** в левом меню платформы.
  </Step>
  <Step title="Нажмите кнопку создания">
    В левой панели нажмите иконку **«+»** рядом с заголовком **Folders**.
  </Step>
  <Step title="Введите название">
    Укажите понятное название -- например, «Прайсы», «Правила», «Описания номеров». Папка появится в дереве.
  </Step>
</Steps>

Вы можете создавать папки внутри папок для более глубокой организации, например: **Прайсы → Лето 2026**, **Прайсы → Зима 2026**.

### Фильтрация по сотруднику

Над таблицей есть выпадающий список **All agents**. Выберите конкретного сотрудника -- и увидите только те источники, которые к нему подключены. Удобно, когда нужно понять, какие знания использует конкретный сотрудник.

## Вкладка «База знаний» -- настройка конкретного сотрудника

Откройте нужного ИИ-сотрудника → вкладка **База знаний**. Здесь вы выбираете, какие источники из общего хранилища подключить к этому сотруднику.

### Интерфейс

Экран разделён на две части:

**Левая панель -- дерево папок.** Все папки с источниками знаний. У каждой папки есть **чекбокс**:

| Состояние чекбокса | Значение |
|---|---|
| Отмечен | Источники из этой папки подключены к текущему сотруднику |
| Не отмечен | Источники не используются |
| Частично отмечен | Часть источников внутри подключена, часть нет |

Просто поставьте или снимите галочку, чтобы подключить или отключить целую папку.

**Правая панель -- таблица файлов.** При выборе папки показывает таблицу **Knowledge files**:

| Столбец | Что показывает |
|---|---|
| **Access** | Чекбокс -- подключён ли источник к этому сотруднику |
| **Название** | Имя файла, URL или текстового источника |
| **Type** | Тип источника (Text, Folder и др.) |
| **Updated** | Дата последнего обновления |

Над таблицей -- **хлебные крошки** (breadcrumbs) для навигации по папкам.

### Подключение источников к сотруднику

<Steps>
  <Step title="Откройте вкладку «База знаний»">
    Перейдите в настройки нужного сотрудника → вкладка **База знаний**.
  </Step>
  <Step title="Найдите нужные источники">
    В левой панели найдите папку с нужными источниками.
  </Step>
  <Step title="Поставьте галочку">
    Отметьте чекбокс напротив папки или отдельного файла.
  </Step>
  <Step title="Сохраните">
    Нажмите **Сохранить**. Один и тот же источник можно подключить к нескольким сотрудникам.
  </Step>
</Steps>

<Tip>
  Организуйте источники по папкам -- например, одна папка на сотрудника или на тему. Так проще управлять доступом, когда источников станет много.
</Tip>

## Четыре способа добавить источник знаний

Находясь в нужной папке (в любом из двух разделов), нажмите кнопку **Добавить источник**. Доступны четыре варианта:

### 1. Загрузить файлы

Загрузка документов с компьютера. Поддерживаемые форматы:

| Формат | Описание |
|---|---|
| **PDF** | Документы, прайс-листы, каталоги |
| **DOCX** | Текстовые документы Word |
| **TXT** | Простые текстовые файлы |
| **CSV** | Таблицы с данными |

**Когда использовать:** есть готовые документы -- прайсы, регламенты, описания услуг, инструкции.

<Tip>
  Для лучшего результата используйте структурированные документы с заголовками и списками. ИИ лучше работает с хорошо организованным текстом, чем со сплошным потоком символов.
</Tip>

### 2. Текст

Ручной ввод или вставка текста прямо в платформу.

| Поле | Описание |
|---|---|
| **Название** (необязательно) | Понятное имя для источника |
| **Содержимое** | Сам текст -- вставьте или введите вручную |

**Когда использовать:** нужно быстро добавить краткую информацию -- адрес, контакты, режим работы, особые условия.

### 3. YouTube видео

Импорт транскрипции видео с YouTube. Укажите **URL** -- платформа автоматически извлечёт текст из субтитров.

**Когда использовать:** полезная информация содержится в видеообзорах, презентациях или обучающих роликах.

<Warning>
  Импорт работает только с видео, у которых есть субтитры -- автоматические или загруженные автором.
</Warning>

### 4. Веб-страница

Импорт содержимого по URL. Поддерживает обычные сайты, **Google Docs** и **Google Sheets**.

| Параметр | Описание |
|---|---|
| **URL** | Адрес страницы |
| **Режим парсинга** | **Одна страница** -- только указанный URL. **Весь сайт** -- рекурсивный обход всех страниц |

**Когда использовать:** информация уже опубликована на сайте или в Google-документе.

<Tip>
  Для импорта из Google Docs или Google Sheets просто вставьте ссылку -- система распознает формат автоматически. Для приватных документов потребуется предоставить доступ сервисному аккаунту платформы -- адрес указан в диалоге импорта.
</Tip>

## Обновление знаний

Загруженные источники не обновляются автоматически. Если прайс-лист изменился -- сотрудник продолжит использовать старые данные.

<Steps>
  <Step title="Найдите устаревший источник">
    Перейдите в раздел **Знания** и найдите нужный файл.
  </Step>
  <Step title="Удалите старую версию">
    Через меню действий (три точки) удалите устаревший источник.
  </Step>
  <Step title="Загрузите новую версию">
    Загрузите новый файл или заново импортируйте страницу.
  </Step>
</Steps>

<Warning>
  При удалении источника он пропадает у **всех** сотрудников, которые его использовали. Если нужно убрать источник только у одного сотрудника -- не удаляйте его, а снимите чекбокс во вкладке **База знаний** этого сотрудника.
</Warning>

## Сводная таблица действий

| Действие | Где выполняется |
|---|---|
| Загрузить новый источник | **Знания** (левое меню) или **База знаний** (вкладка в сотруднике) |
| Подключить источник к сотруднику | **База знаний** (вкладка в сотруднике) -- чекбоксами |
| Посмотреть все источники по всем сотрудникам | **Знания** (левое меню) |
| Удалить источник из системы | **Знания** (левое меню) |
| Отключить источник от сотрудника (не удаляя) | **База знаний** (вкладка в сотруднике) -- снять чекбокс |

## Примеры использования

<Accordion title="Отель -- централизованная база">
  Отель загрузил источники в раздел **Знания** и разложил по папкам:

  ```
  All knowledge
  ├── Agents
  │   ├── Консьерж (Telegram)
  │   └── Поддержка (виджет на сайте)
  ├── Общие документы
  │   ├── Правила проживания.pdf
  │   ├── Прайс-лист 2026.docx
  │   └── Описание номеров.txt
  ├── Рестораны и спа
  │   ├── Меню ресторана.pdf
  │   └── Услуги спа.txt
  └── Трансфер
      └── Маршруты и цены.txt
  ```

  Папки «Общие документы» и «Рестораны и спа» подключены к обоим сотрудникам. «Трансфер» -- только к Консьержу. Когда изменились цены на спа -- менеджер удалил старый файл и загрузил новый. Оба сотрудника сразу получили актуальные данные.
</Accordion>

<Accordion title="Интернет-магазин -- разделение по направлениям">
  Два сотрудника: «Консультант по товарам» и «Служба поддержки».

  - **Каталог** -- описания товаров, характеристики, наличие. Подключён только к Консультанту.
  - **Политики** -- возвраты, доставка, гарантия. Подключён только к Службе поддержки.
  - **Общее** -- контакты, режим работы, акции. Подключён к обоим.

  Когда появился новый товар -- файл добавили в «Каталог». Консультант сразу может о нём рассказать, а Служба поддержки его даже не видит.
</Accordion>

<Accordion title="Онлайн-школа -- загрузка расписания">
  Школа загрузила расписание занятий в CSV и подключила к сотруднику-куратору. Студенты спрашивают «Когда следующее занятие по маркетингу?» -- куратор находит ответ в таблице. Расписание обновляется раз в месяц: администратор удаляет старый CSV и загружает новый.
</Accordion>

## Рекомендации

- **Понятные имена файлов.** «Прайс-лист 2026 март.pdf» -- сразу ясно. «Document_final_v3(2).pdf» -- нет.
- **Не загружайте всё подряд.** Каждый лишний источник -- это контекст, в котором сотрудник ищет ответ. Больше нерелевантных документов -- больше шансов на неточный ответ.
- **Структурируйте документы.** Заголовки, списки, таблицы. Длинный документ без структуры лучше разбить на несколько тематических файлов.
- **Проверяйте после загрузки.** Откройте тестовый чат и задайте вопросы по загруженной информации.
- **Следите за актуальностью.** Устаревшие данные -- это неправильные ответы. Сотрудник не знает, что информация устарела.

## Частые вопросы

<Accordion title="Сколько источников можно добавить?">
  Ограничений по количеству нет. Но чем больше источников, тем больше токенов расходуется на поиск -- добавляйте только то, что действительно нужно сотруднику.
</Accordion>

<Accordion title="Данные обновляются автоматически?">
  Нет. Все знания -- статические. Если исходный документ изменился, нужно удалить старый источник и загрузить новую версию. Автоматической синхронизации нет.
</Accordion>

<Accordion title="Можно ли использовать один источник для нескольких сотрудников?">
  Да. Источник хранится в разделе **Знания** и подключается к любому количеству сотрудников через чекбоксы во вкладке **База знаний** каждого сотрудника.
</Accordion>

<Accordion title="Что произойдёт, если удалить источник в разделе «Знания»?">
  Он пропадёт у всех сотрудников, к которым был подключён. Если нужно убрать источник только у одного сотрудника -- снимите чекбокс во вкладке **База знаний**, не удаляя файл.
</Accordion>

<Accordion title="Сотрудник не использует данные из базы знаний. Что делать?">
  Проверьте три вещи: (1) источник подключён -- чекбокс стоит во вкладке **База знаний**; (2) информация хорошо структурирована -- заголовки, списки; (3) в [Персоне](/ru/guides/persona) есть указание «отвечай на основе данных из базы знаний, не выдумывай».
</Accordion>

<Accordion title="Влияет ли структура папок на работу сотрудника?">
  Нет. Папки -- инструмент для вашего удобства. Сотрудник ищет по всем подключённым источникам, независимо от расположения в папках.
</Accordion>

<Accordion title="Что лучше: загрузить файл или вставить текст?">
  Если информация уже есть в документе -- загружайте файл. Если нужна краткая справка (адрес, контакты, правило) -- используйте текст. Для актуальной информации с сайта -- импортируйте веб-страницу.
</Accordion>
