Профили — это карточки клиентов, которые ИИ-сотрудник заполняет **автоматически** прямо во время диалога. Никаких ручных действий не нужно: пока сотрудник общается с клиентом, он параллельно извлекает нужные данные и записывает их в структурированную таблицу.

Вы сами решаете, что именно собирать. Имя и язык — по умолчанию. Всё остальное — поля, которые вы настраиваете под свои задачи.

## Настройка полей профиля

Откройте нужного ИИ-сотрудника → вкладка **Профили**.

### Режим обновления

Первое, что нужно выбрать — когда сотрудник обновляет профиль.

| Режим | Когда обновляется | Когда использовать |
|---|---|---|
| **Realtime** | После каждого сообщения клиента | Если сотруднику нужно использовать данные прямо сейчас — например, обращаться к клиенту по имени или учитывать предпочтения в том же разговоре |
| **Плановая** | По расписанию (например, раз в 24 часа) | Если данные нужны только для аналитики и отчётности |

При выборе планового режима появляются дополнительные настройки: интервал, час запуска и часовой пояс. Там же отображается прогноз расхода токенов — сколько будет потрачено в сутки и в месяц.

<Tip>
  Если не знаете, что выбрать — используйте **плановое обновление раз в 24 часа**. Этого достаточно для большинства сценариев и значительно экономит бюджет по сравнению с Realtime.
</Tip>

### Типы полей

Каждое поле имеет тип, который определяет, может ли сотрудник обновить его значение.

| Тип | Поведение | Пример |
|---|---|---|
| **Variable** | Обновляется при каждом новом упоминании | Язык, предпочтения, статус, текущий интерес |
| **Constant** | Фиксируется один раз и больше не меняется | Номер договора, дата рождения, тип клиента |

<Warning>
  Константу нельзя изменить после первого заполнения. Если не уверены — выбирайте **Variable**.
</Warning>

---

### Параметры поля

Каждое поле состоит из четырёх параметров:

| Параметр | Что писать |
|---|---|
| **Название** | Понятное имя поля: «Имя», «Язык», «Статус обращения» |
| **Описание** | Инструкция для ИИ — когда и что заполнять. Это **самый важный параметр** |
| **Ключ** | Генерируется автоматически, менять не нужно |
| **Тип** | Variable или Constant |

<Warning>
  **Описание поля — ключевой момент.** Пишите его **от лица клиента**: не «когда клиент называет язык», а «язык, на котором я общаюсь». Чем точнее описание — тем точнее сотрудник понимает, в какой момент заполнять поле.

  Также укажите формат ответа. Например: «да/нет» или «выбери один из вариантов: русский, английский, другой». Без этого сотрудник может записать произвольный текст, который будет сложно анализировать.
</Warning>

### Как добавить поле

<Steps>
  <Step title="Откройте вкладку «Профили»">
    Перейдите в карточку нужного ИИ-сотрудника и выберите вкладку **Профили**.
  </Step>
  <Step title="Добавьте новое поле">
    Нажмите кнопку добавления нового поля.
  </Step>
  <Step title="Введите название">
    Задайте понятное имя — например, «Имя», «Язык», «Бюджет».
  </Step>
  <Step title="Заполните описание">
    Подробно опишите **от лица клиента**, когда и что заполнять. Укажите формат ответа.
  </Step>
  <Step title="Выберите тип">
    **Variable** — если значение может меняться. **Constant** — если фиксируется один раз.
  </Step>
  <Step title="Сохраните">
    Повторите для каждого нужного поля. Количество полей не ограничено.
  </Step>
</Steps>

<Tip>
  Рекомендуется добавлять только те поля, которые действительно нужны — это повышает точность заполнения и экономит токены.
</Tip>

### Примеры полей

<Accordion title="Посмотреть примеры настройки полей">

| Поле | Тип | Описание для ИИ |
|---|---|---|
| Имя | Variable | Моё имя |
| Язык | Variable | Язык, на котором я общаюсь. Выбери один вариант: русский, английский, другой |
| Предпочтения | Variable | Предпочтения или пожелания, которые я упомянул. Обновлять при каждом упоминании |
| Жалоба | Variable | Когда я говорю о любой проблеме. Описать кратко, не более четырёх слов. Обновлять всегда |
| Статус | Variable | Текущий статус обращения. Выбери один вариант: новый, в работе, решён |
| Бюджет | Variable | Когда я говорю о бюджете или сумме. Записать число |
| Номер договора | Constant | Номер договора или бронирования, который я назвал |
| Резюме | Variable | Два предложения о чём диалог. Смотреть по всей истории. Всегда обновлять |

Это примеры — набор полей полностью зависит от вашего бизнеса и того, что важно отслеживать.

</Accordion>

## Аналитика профилей

Раздел **«Профили»** в левом меню платформы — это аналитический инструмент. Здесь хранятся заполненные карточки всех клиентов по всем сотрудникам.

### Сводные показатели

В верхней части страницы отображаются ключевые метрики:

| Показатель | Что показывает |
|---|---|
| **Всего профилей** | Общее количество клиентов за всё время |
| **За эту неделю** | Новые профили за текущую неделю |
| **За этот месяц** | Новые профили за текущий месяц |
| **Заполненность** | Средний процент заполненности полей — насколько полно сотрудник извлекает данные |

### Таблица профилей

Каждая строка — один клиент. Стандартные столбцы: профиль, агент, Chat ID, имя, резюме, язык. Плюс все пользовательские поля, которые вы настроили.

Возможности таблицы:

- **Фильтрация по агенту** — смотрите данные отдельного сотрудника или филиала
- **Поиск** по имени, содержимому полей, любому тексту
- **Переход в чат** — нажмите на Chat ID, чтобы открыть полную историю диалога
- **Управление столбцами** — скрывайте ненужные колонки
- **Режимы отображения** — переключайтесь между видами **Карточки** и **Таблица**

<Tip>
  Если поле не заполняется — проверьте его описание в настройках сотрудника. Скорее всего, ИИ не понимает, когда именно нужно записывать данные.
</Tip>

## Использование в Персоне

Данные из профилей можно использовать прямо в инструкции сотрудника — через **переменные профиля** в разделе [Персона](/ru/guides/persona). Например, если известно имя клиента, сотрудник может обращаться к нему персонально прямо во время диалога.

Это позволяет создавать по-настоящему персонализированное общение: учитывать предпочтения, адаптировать стиль и предлагать релевантные решения на основе собранных данных.

## Частые вопросы

<Accordion title="Realtime дороже планового?">
  Да, значительно. Realtime обновляет профиль после каждого сообщения и расходует заметно больше токенов. Плановый режим обрабатывает данные пакетами — это экономичнее.
</Accordion>

<Accordion title="Можно ли изменить значение константы?">
  Нет. В этом и суть типа Constant — значение фиксируется один раз и не меняется. Если не уверены, лучше выбрать Variable.
</Accordion>

<Accordion title="Профили общие для всех агентов?">
  Нет, каждый сотрудник ведёт собственные профили клиентов.
</Accordion>

<Accordion title="Сколько полей можно добавить?">
  Количество не ограничено, но рекомендуется добавлять только действительно нужные поля — это повышает точность и снижает расход токенов.
</Accordion>
